# AGENTS.md

## Permanent Development Standard for Edunancial

Repository: `1psiteon1962-stack/edunancial-site`

This document is the permanent operating manual for all GitHub Copilot agents and all future coding agents working on any Edunancial repository.

It applies first to the North American site and is intended to become the global standard for every future regional deployment.

---

## Mission

Every design decision, every feature, every page, and every line of code must support Edunancial's mission:

Financial literacy provides the foundation.

Financial competency is built through disciplined action.

Edunancial exists to help people build practical financial knowledge they can confidently apply to improve their financial decision-making throughout their lives.

We help people build financial competency so they can pursue greater financial independence and greater opportunities to improve their financial circumstances for themselves, their families, and future generations.

---

## Core Development Philosophy

- Substance before appearance.
- Education before marketing.
- Practical application before theory.
- Simplicity before complexity.
- Long-term trust before short-term conversion.

### Operating interpretation

- Every page should educate.
- Every feature should solve a real problem.
- Every interaction should increase user confidence.
- Avoid unnecessary animations, gimmicks, marketing fluff, or empty buzzwords.

---

## Member-First Standard

Before implementing any feature, ask:

- How does this help the member?
- How does this improve financial competency?
- Does it solve a real problem?
- Would the member clearly understand its value?

If the answer is unclear, redesign the feature.

---

## Global Design Standard

All future regional sites shall follow one architecture.

Regions include:

- North America
- Latin America
- Caribbean
- Europe
- Africa
- Asia
- Middle East
- Oceania

Differences between regions should primarily consist of:

- Language
- Regional pricing
- Payment providers
- Country-specific legal disclosures
- Local educational examples

Everything else should remain architecturally consistent.

---

## Language Standard

The language selector must become one of the most visible controls on every page.

- Do **not** hide language selection behind an ambiguous icon.
- Replace the current navigation arrangement with a clearly identifiable **Language** button or **Language selector**.
- Users should immediately understand how to change languages.

Changing languages must instantly translate:

- Navigation
- Buttons
- Forms
- Membership plans
- Mission
- Courses
- Books
- AI Coach
- FAQ
- Contact
- Checkout
- Legal pages
- Error messages
- Success messages
- Every visible piece of text

No page shall remain partially translated.

No mixed-language pages are acceptable.

---

## Mobile First

Every feature shall be designed for mobile before desktop.

Required device/testing targets:

- Android
- iPhone
- Tablet
- Desktop
- Landscape
- Portrait

---

## Accessibility

All work must meet WCAG standards.

Minimum requirements:

- Keyboard navigation
- Screen reader compatibility
- Proper contrast
- Proper button sizing
- Visible focus states
- Accessible forms

---

## Membership Experience

Members should immediately understand:

- What they receive
- Why it matters
- How it helps
- What they should do next

The website should reduce confusion rather than create it.

---

## Content Standard

Every educational page should answer:

- What is it?
- Why does it matter?
- How is it used?
- What mistakes should be avoided?
- How can someone apply this knowledge?

Knowledge must become action.

---

## Global Pricing

The platform must:

- Support country-specific pricing
- Support local currencies
- Support purchasing power adjustments where appropriate
- Support future country additions without requiring architectural redesign

---

## Quality Standard

No feature is complete until all of the following are done:

- Code completed
- Tests passed
- Mobile verified
- Desktop verified
- Accessibility verified
- Translations verified
- Pull Request created
- Pull Request approved
- Merged into `main`
- Production deployed
- Production verified

---

## Repository Governance

Follow these documents without exception:

- `REPOSITORY_WORKFLOW_POLICY.md`
- `REPOSITORY_GOVERNANCE_POLICY.md`

---

## End-of-Task Requirement

At the completion of every task, the agent shall provide:

1. Summary of work completed
2. Files modified
3. Pull Request information
4. Merge status
5. Deployment status
6. Production verification
7. Recommended next highest-priority task

---

## Long-Term Objective

Build the world's most practical financial education platform.

Measure success not by the number of pages created, but by the number of people who become more capable, more confident, and better equipped to make sound financial decisions.
